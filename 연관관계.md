###연관관계
```
방향 : 단방향(회원 -> 팀 또는 팀 -> 회원), 양방향(회원 -> 팀 그리고 팀 -> 회원)  
* 방향은 객체 관계에 존재하며 테이블에서는 항상 양방향 상태

다중성 : 다대일(N:1), 일대다(1:N), 일대일(1:1), 다대다(N:N)
회원, 팀을 회원 기준으로 여러 회원은 하나의 팀에 소속될 수 있으므로 N:1 관계
반대로 팀을 기준으로 하면 1:N 관계가 된다

연관관계 주인 : 객체를 양방향 연관관계로 만들 경우 주인을 정해야 한다
```

####단방향 연관관계 다대일(N:1)
```
객체 연관관계

    객체는 필드로 연관관계를 맺음
    Member 클래스에 Team 필드를 호출하는 필드는 있어서 조회 가능
    Team 에서 Member를 호출하는 필드가 없어서 조회가 불가능

테이블 연관관계

    테이블은 외래 키로 연관관계를 맺음
    테이블은 양방향 관계이므로 외래키를 사용하여 필요한 곳 에서 조인하여 관계를 맺는다

객체 연관관계와 테이블 연관관계의 차이

    객체의 참조를 통한 연관관계는 항상 단방향이며 양방향의 연관관계를 맺고 싶은경우
    반대쪽에서 참조하기 위한 필드를 생성해주어야 한다 그러나 정확한 표현은
    서로 다른 2개의 단방향 관계라고 할 수 있다

객체 연관관계 vs 테이블 연관관계 정리

    객체는 참조(주소)로 연관관계를 맺는다 (단방향)
    테이블은 외래 키로 연관관계를 맺는다 (양방향)
    객체를 양방향으로 참조하려면 단방향 연관관계를 2개 생성
```
